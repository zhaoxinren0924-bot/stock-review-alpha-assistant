# Stock Review Alpha Assistant — 项目宪法

## 项目定位

A股基本面复盘智能助手。为有一定投资经验、但不想和量化拼短线速度的 A 股个人投资者，提供一个 AI 驱动的基本面假设管理与复盘平台。

## 目标用户

有 1-5 年 A 股投资经验的个人投资者，资金规模 5-50 万，通过基本面、行业逻辑、财报、公告和长期趋势做决策。不追求高频交易，也不相信 AI 能直接告诉自己明天涨跌。

## 核心价值

1. **第一次打开**：把每只股票的买入理由整理成清晰的基本面假设，而不是散落在脑子、微信、Excel 和券商 App 里
2. **持续使用**：系统持续跟踪财报、公告、新闻，提醒哪条投资假设被增强、削弱或需要复盘
3. **推荐给别人**：不是又一个行情软件，而是管理自己的投资逻辑、复盘错误和长期持仓纪律的工具

## 必须做的 5 件事

1. **股票池与持仓管理** — 添加关注公司、记录持仓成本/数量/日期、展示基础行情/仓位/浮盈亏/下次复盘时间
2. **基本面假设管理** — 围绕生意质量、财务质量、成长逻辑、估值水平、风险变化、催化事件六框架建立可验证假设
3. **AI 复盘与成果生成** — 右侧 AI 对话窗口，自然语言输入转化为结构化成果（假设/风险点/检查项/复盘记录/规则提醒）
4. **公告/新闻影响判断** — 自动跟踪关注股票动态，AI 筛选有价值信息并判断影响方向（增强/削弱/风险/中性）
5. **个人规则与错误画像** — 投资规则检查、复盘归因、常见错误统计与提醒

## 绝不做的 3 件事

1. **不做短线荐股和买卖点预测** — 不告诉用户明天涨不涨，不推荐股票，不替用户做最终买卖决策
2. **不做普通行情软件的复制品** — 不堆 K 线、盘口、资讯流、社区热帖和题材榜
3. **不做高成本复杂系统** — MVP 不接入付费昂贵数据源，不做券商实盘交易接入，不做需要用户运维的自建服务器

## 技术栈

- **后端**: FastAPI + SQLAlchemy + SQLite + Pandas
- **前端**: React + Vite + TypeScript + Tailwind CSS + shadcn/ui
- **AI**: Claude API (Anthropic)
- **数据源**: a-stock-data (A股7层/28端点/13源)
- **部署**: Render (staging) / 本地运行 (production)

## 不变量

1. **基本面优先** — 始终服务于中长期基本面判断，不和量化/高频/短线情绪交易竞争
2. **AI 必须生成可保存成果** — 每次 AI 介入都要转化为投资假设/复盘记录/规则/检查项/影响判断，不是只输出一段看完就忘的文字
3. **用户自己做最终决策** — 系统只帮助整理信息/发现变化/复盘错误，不替用户下结论/不承诺收益
4. **数据可迁移、本地/导出友好** — 持仓/假设/笔记/规则/复盘记录必须易于导出，不能被平台锁死
5. **界面简洁、信息密度高** — 页面优先展示状态/证据/动作，少用长篇 AI 文本，避免制造新的信息噪音
6. **Pandas 只做 ETL，不做持久层** — 清洗完立即写入 SQLite，不在内存驻留
7. **单命令启动** — 用户只需 `python main.py`，无需配置多个服务

```

│   ├── tests/         # 测试
│   └── requirements.txt
├── frontend/
│   └── src/           # React 前端
```

## 数据分层治理

| 层级 | 来源 | 处理 | 存储 |
|---|---|---|---|
| 原始层 | a-stock-data 28端点 | 拉取 | - |
| 清洗层 | Pandas DataFrame | 标准化/去重/质量评分 | - |
| 结构化层 | SQLAlchemy ORM | 关联/分类 | SQLite |
| AI上下文 | Prompt构建 | 影响判断 | SQLite impacts 表 |

## 成功指标

1. **自己连续 2 周每个交易日打开使用** — 完成查看持仓假设/阅读影响事件/记录复盘中的任意一个动作
2. **至少 3 个朋友愿意录入自己的股票并持续使用 7 个交易日**
