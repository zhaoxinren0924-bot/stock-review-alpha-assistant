# Investment Themes — 投资主题模板设计

> **核心洞察**：散户做基本面研究的投资单位不是"行业"，是"主题"。用户不是想分析"通信设备行业"，而是想判断"AI 算力这条线还能不能继续"。

---

## 设计原则

1. **不按教科书行业分类**：不按申万一级/二级行业做模板。
2. **按投资主线组织**：围绕用户真实关注的产业趋势、政策方向、资金逻辑。
3. **一只股票可多主题**：中际旭创可以同时属于"AI 算力链"和"光通信"两个主题。
4. **主题 = 核心假设 + 关键指标 + 关键风险**：不是静态标签，是可验证的投资逻辑框架。

---

## 第一版默认主题（5 个）

### 1. AI 算力链

**适合标的**：中际旭创、新易盛、工业富联、浪潮信息、寒武纪、海光信息

**核心假设**：
- AI 资本开支持续增长
- 海外订单保持高景气
- 业绩增速能够消化估值
- 公司在产业链中具备竞争优势

**关键指标**：
- 营收增速、净利润增速
- 毛利率、海外收入占比
- 存货、应收账款
- 资本开支、PE(TTM)

**关键风险**：
- 股价涨幅领先业绩兑现
- 大客户订单波动
- 毛利率下行
- 技术路线变化

---

### 2. 半导体国产替代

**适合标的**：北方华创、中芯国际、韦尔股份、兆易创新、澜起科技

**核心假设**：
- 国产替代进度持续推进
- 下游需求进入复苏周期
- 库存去化完成，毛利率修复
- 政策支持力度不减

**关键指标**：
- 营收增速、净利润增速
- 毛利率、研发投入占比
- 库存周转天数
- 订单能见度、PE(TTM)

**关键风险**：
- 周期复苏不及预期
- 库存压力持续
- 技术受限（先进制程）
- 高估值透支预期

---

### 3. 机器人 / 智能制造

**适合标的**：绿的谐波、埃斯顿、汇川技术、鸣志电器、三花智控

**核心假设**：
- 人形机器人产业进入量产前夜
- 核心零部件国产化率提升
- 订单快速增长，客户导入顺利
- 规模效应带来成本下降

**关键指标**：
- 营收增速、净利润增速
- 毛利率、客户集中度
- 产能利用率
- 订单金额、PE(TTM)

**关键风险**：
- 主题炒作领先基本面
- 量产进度不及预期
- 价格战压缩利润
- 下游需求不足

---

### 4. 创新药 / 医疗器械

**适合标的**：恒瑞医药、百济神州、迈瑞医疗、药明康德、联影医疗

**核心假设**：
- 研发管线进入收获期
- 核心产品临床/商业化进展顺利
- 出海逻辑逐步兑现
- 集采影响边际减弱

**关键指标**：
- 研发费用、管线进展
- 商业化收入增速
- 毛利率、费用率
- 海外收入占比、PE(TTM)

**关键风险**：
- 研发失败或临床不及预期
- 集采降价超预期
- 出海受阻（地缘政治）
- 现金流压力

---

### 5. 高股息 / 央国企红利

**适合标的**：中国移动、中国神华、长江电力、招商银行、中国海油

**核心假设**：
- 分红政策稳定或提升
- 盈利和现金流具备可持续性
- 估值处于合理或偏低水平
- 利率下行环境利好高股息资产

**关键指标**：
- 股息率、分红率
- ROE、净利润增速
- 经营性现金流
- 资本开支计划、PB

**关键风险**：
- 分红下降或政策变化
- 盈利下滑
- 资本开支上升挤压分红
- 估值过度拥挤（资金抱团）

## 数据模型设计

### 主题模板表

```sql
investment_themes
- id
- theme_code              -- ai_compute_chain / semiconductor_localization / ...
- theme_name              -- "AI 算力链"
- theme_category          -- growth / value / cyclical / defensive
- core_metrics            -- JSON，关键指标列表
- core_hypotheses         -- JSON，默认核心假设
- core_risks              -- JSON，默认关键风险
- description             -- 主题描述
- is_active               -- 是否启用
- sort_order              -- 排序
- created_at
```

### 股票-主题关联表

```sql
stock_themes
- id
- stock_code
- theme_id
- is_primary              -- 是否为主要主题
- user_override           -- 用户自定义（覆盖默认）
- created_at
```

一只股票可以属于多个主题，其中一个是主要主题。

## 与假设管理的关联

当用户添加股票时：

1. 系统根据股票代码匹配默认主题（如中际旭创 → AI 算力链）。
2. 自动预填该主题的核心假设、关键指标、关键风险作为 Hypothesis 的初始内容。
3. 用户可以修改、删除、添加自己的假设。
4. 主题只提供"起点模板"，最终假设由用户定义。

示例：

```json
{
  "stock_code": "300308",
  "theme": "ai_compute_chain",
  "auto_populated": [
    {
      "category": "growth_logic",
      "content": {
        "title": "AI 资本开支持续增长",
        "summary": "海外云厂商 Capex 维持高位，带动光模块需求",
        "hypotheses": [
          { "label": "H1", "text": "海外 AI 资本开支持续增长", "status": "active" },
          { "label": "H2", "text": "公司在 800G 光模块具备先发优势", "status": "active" }
        ]
      }
    },
    {
      "category": "risk_changes",
      "content": {
        "title": "估值与订单风险",
        "summary": "股价涨幅较大，需关注业绩兑现节奏",
        "risks": [
          { "text": "股价涨幅领先业绩兑现", "affected_hypothesis_labels": ["H1"] },
          { "text": "大客户订单波动", "affected_hypothesis_labels": ["H2"] }
        ]
      }
    }
  ]
}
```

## MVP 策略

**Phase 1**：不做主题模板系统。
- Hypothesis 的 content 完全由用户手动录入。
- 主题模板作为设计预留，不实现。

**Phase 2**：接入 a-stock-data 后，实现：
- 5 个默认主题模板
- 股票-主题自动匹配
- 添加股票时自动预填假设

**Phase 3**：
- 用户自定义主题模板
- 主题级别的批量复盘
- 跨主题比较（如"AI 算力链 vs 高股息"的配置建议）

---

## 与行业分类的关系

| 维度 | 用途 | 示例 |
|---|---|---|
| **申万行业** | 数据源分类、行情对比 | 通信设备 |
| **投资主题** | 假设模板、复盘框架 | AI 算力链 |
| **用户逻辑** | 个性化假设、自定义指标 | 海外订单 + 毛利率 |

三者不互斥，共存于系统中。
