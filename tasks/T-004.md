## 目标(What)

一句话: 实现 Hypothesis、Position、Review、CheckItem 的完整 CRUD API。

具体:
- **Hypothesis API**：`GET/POST /api/v1/stocks/{code}/hypotheses`、`PUT/DELETE /api/v1/hypotheses/{id}`。
  - content 为 JSON 字段，6 种框架结构（见 `docs/api-contract.md` §4.1）。
  - 创建时自动查找/创建 InvestmentCase。
- **Position API**：`GET/POST /api/v1/stocks/{code}/position`、`PUT/DELETE /api/v1/positions/{id}`。
  - 单条汇总持仓语义（MVP 不录逐笔交易）。
- **Review API**：`GET/POST /api/v1/stocks/{code}/reviews`、`PUT/DELETE /api/v1/reviews/{id}`。
  - review_type: `quarterly` / `event`。
- **CheckItem API**：`GET/POST /api/v1/stocks/{code}/check-items`、`PUT/DELETE /api/v1/check-items/{id}`。
  - status: `pending` / `done` / `skipped`。
  - 独立的 CheckItem 模型，不复用 Review 表。
- Pydantic schemas、路由、pytest 测试覆盖。

## 注意事项

- AI 必须生成可保存成果是不变量：Hypothesis content JSON 结构需要兼容 AI 生成的假设内容。
- Pandas 只做 ETL，不做持久层是不变量：API 层直接使用 SQLAlchemy ORM。
- 单命令启动是不变量：新增路由后确认 `python main.py` 仍能正常启动。
- 命名规范：后端返回 `snake_case`，前端负责转换。
- `DELETE /api/v1/stocks/{code}` 级联删除所有用户私有数据，使用 SQLAlchemy cascade。
- 详细 API 定义见 `docs/api-contract.md` §4。
