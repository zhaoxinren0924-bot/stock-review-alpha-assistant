## 目标(What)

一句话: 接入 Claude API，实现 AI 对话和结构化成果生成。

具体:
- **后端**：
  - 创建 `POST /api/v1/ai/chat` 路由。
  - Prompt 工程：将当前股票的六框架假设、持仓信息作为上下文传入 Claude。
  - 要求 Claude 返回结构化 JSON（含 `reply` 文本 + `actions` 数组）。
  - 解析 Claude 回复，提取结构化成果（固定 4 种 action 类型）。
  - 创建 `POST /api/v1/ai/actions/apply` 路由：验证 payload schema，执行对应的数据库操作。
- **前端**：
  - 连接 `/api/v1/ai/chat`，替换 mock 消息。
  - 展示 AI 返回的待保存成果卡片（T-009 已实现 UI）。
  - 实现保存/编辑/忽略操作的 API 调用。

---

## 不在范围内

明确**不**做的事:
- 不做流式输出（SSE），先使用一次性响应。
- 不做多轮对话上下文管理（单轮问答即可，history 数组预留）。
- 不做 AI 的错误恢复（超时/异常时返回固定错误提示）。
- 不接入外部新闻/公告数据源（仅使用已有假设作为上下文）。
- action 不持久化（无 ai_suggested_actions 表）。

## 注意事项

- AI 必须生成可保存成果是不变量：Prompt 中必须明确要求 Claude 返回结构化 JSON（包含 `analysis`、`actions` 数组），不能只返回纯文本分析。
- 用户自己做最终决策是不变量：Prompt 中要求 Claude 使用中性语言，禁止给出买入/卖出建议，只允许说"需要关注"、"建议复盘"。
- 基本面优先是不变量：Prompt 上下文优先传入六框架假设，而不是行情数据。
- API Key 安全：`.env` 文件加入 `.gitignore`，生产环境使用 Render 环境变量。
- Claude 模型选择：使用 `claude-sonnet-4-6` 或 `claude-opus-4-7`，平衡成本与效果。
- 上下文长度控制：单股票六框架假设 + 持仓信息通常不超过 4K tokens，远小于上下文窗口限制。
- action 类型固定 4 种，每种有预定义 payload schema（`docs/api-contract.md` §5.2）。
