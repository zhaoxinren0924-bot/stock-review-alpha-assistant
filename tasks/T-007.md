## 目标(What)

一句话: 实现六框架卡片的完整展示，每类卡片有独立的 content 渲染模板。

具体:
六张卡片放入 2 列 × 3 行网格，每张卡片内部结构不同：

1. **生意质量**：核心判断段落 + 证据标签（Badge 列表）。
2. **财务质量**：关键指标表格（ROE/不良率/拨备覆盖率/AOE），每个指标带小型状态标记。
3. **成长逻辑**：核心假设列表（H1/H2），每条带状态标记 + "更新假设"按钮。
4. **估值水平**：指标数值横排（PB/PE/股息率）+ AI 结论段落。
5. **风险变化**：主要风险 bullet list + 影响假设标签 + "加入复盘"按钮。
6. **催化事件**：最新事件列表 + 影响方向箭头（增强/削弱/影响 H1/H2）+ "查看影响"按钮。

所有数据从 Hypothesis API 获取，按 `category` 分组展示。

## 不在范围内

- "创建检查项"、"更新假设"、"加入复盘"、"查看影响"按钮不产生实际行为（仅视觉或 console.log）。
- 不实现卡片的展开/折叠。
- 不实现假设的展开详情（仅展示标题和摘要）。

## 注意事项

- 六框架卡片内部结构各不相同，不要试图用统一模板硬套。
- content JSON 结构见 `docs/api-contract.md` §4.1，所有字段可选（除 title/summary 外）。
- 状态标签必须复用 `StatusBadge`（`docs/design-system.md` §4.4）。
- 基本信息面优先是不变量：卡片内容必须围绕基本面逻辑组织。
- 用户自己做最终决策是不变量：卡片只呈现信息，不加买卖建议文案。
- 成长逻辑假设使用 H1/H2 编号体系，后续 AI 对话中会引用。
- 卡片圆角统一 8px（`rounded-lg`），无阴影。
